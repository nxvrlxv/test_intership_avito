# PR Reviewer Assignment Service (Fall 2025)

Сервис для автоматического назначения ревьюверов на Pull Request по командам.

## Особенности реализации

- FastAPI + SQLAlchemy + Pydantic
- Автоматическое назначение 2 активных участников из той же команды (исключая автора)
- Поддержка переназначения ревьювера
- Запуск одной командой docker compose up
- Для наполнения БД использовать файл seed_db.py

## Проблемы, с которыми столкнулся на этапе решения
- Возникла ошибка маршрутизации: когда пытался обратиться к статическим маршрутам для пользователя (/getReview; /all_users) возникала ошибка 404 User not found. Решение проблемы оказалось очень простым: нужно было сначала описать все статические маршруты, и только после этого описывать динамические (/{user_id})
- не собирался контейнер для БД. Не мог залогиниться на docker hub -> не мог сделать pull дефолтного образа для сборки своего контейнера. Решил сменой настроек конфигурационного файла Docker (необзодимо было установить docker-credential-osxkeychain хелпер и поменять "credsStore": "desktop" на "credsStore": "osxkeychain" ) 
- не собирался контейнер для проекта. Проблема была в неправильном содержании requirements.txt (просто недописал библиотеки)

## Используемый стек
- Python 3.11
- FastAPI
- SQLAlchemy 2.0
- PostgreSQL (в Docker) / SQLite (локально)
- Pydantic v2
- Uvicorn
- Docker + docker-compose

## Инструкция для запуска
- скопировать репозиторий к себе на пк (git clone ссылка_на_репозиторий)
- Запустить docker демон
- docker compose up
